# RIFT — State directory layout, backup, and restore

This document explains what lives under `~/.rift/`, what's irrecoverable if lost, what's cache, and how to move a RIFT install between machines.

---

## The 30-second version

```bash
# Backup what matters (skip caches — they regenerate)
tar -czf rift-backup-$(date +%Y%m%d).tgz \
  ~/.rift/credentials \
  ~/.rift/.env \
  ~/.rift/tokens/ \
  ~/.rift/signal_memory.jsonl \
  ~/.rift/lessons.json \
  ~/.rift/validated_edge.json \
  ~/.rift/bundles/ \
  ~/.rift/recon/ \
  ~/.rift/algo_sessions/

# Restore on a new machine
mkdir -p ~/.rift
tar -xzf rift-backup-YYYYMMDD.tgz -C ~/.rift

# Verify
rift doctor
```

## What lives where

| Path | Contents | Backup priority | Recoverable? |
|---|---|---|---|
| `~/.rift/.env` | AWS keys, optional HL key, proxy URL | **Critical** | No — you wrote these by hand |
| `~/.rift/credentials` | API wallet private key + builder-fee state + agent metadata | **Critical** | Only by re-running `rift auth setup` (creates a fresh agent — costs gas and you lose history) |
| `~/.rift/tokens/` | Authorization tokens for trade gating | **Critical** | Only by re-issuing each token |
| `~/.rift/accepted_disclaimer` | Sentinel file (you accepted the trading risk disclaimer once) | Low | Just accept again on next live command |
| `~/.rift/signal_memory.jsonl` | Append-only log of trade outcomes — drives signal weighting in scout | **High** | No — this is your edge memory |
| `~/.rift/lessons.json` | Post-trade learnings captured manually | High | No — these are your notes |
| `~/.rift/validated_edge.json` | Promoted strategy metadata (which strategies passed validation gates) | Medium | Recoverable via `rift research <strategy>` re-run |
| `~/.rift/bundles/` | Sealed reproducibility manifests for promoted strategies | Medium | Recoverable via re-research |
| `~/.rift/recon/` | Recon trade logs (JSON per session) | Medium | No — historical trade outcomes |
| `~/.rift/algo_sessions/` | Algo daemon final session logs (per-run summaries) | Medium | No — historical session outcomes |
| `~/.rift/algo/` | Live daemon state, pids, current-session snapshots | Low | Regenerated by daemon on next start |
| `~/.rift/data/` | Cached candles, funding, L2 books from sync | Cache | Recoverable via `rift sync` (~1GB download, ~$2 in AWS S3 transfer) |
| `~/.rift/raw/` | Raw S3 download cache (pre-normalized) | Cache | Re-syncable |
| `~/.rift/raw_l2/` | Raw L2 order book download cache | Cache | Re-syncable |
| `~/.rift/reports/` | Generated tearsheets, attribution reports | Cache | Re-generatable via `rift report` / `rift tearsheet` |
| `~/.rift/simulations/` | Old paper-trade session logs (pre-0.1, not actively written) | Skip | Not used by current RIFT |
| `~/.rift/proposals/` | AI-agent trade proposals awaiting approval | Low | Ephemeral by design |
| `~/.rift/workbench/` | User-authored strategy drafts (when using `rift workbench-*`) | High | No — your work |
| `~/.rift/collector.db` / `.log` | Live websocket collector state | Cache | Restarts cleanly |
| `~/.rift/config.json` | Per-install settings (proxy URL, etc.) | Medium | Re-settable via `rift set-proxy`, etc. |

**Critical** = lose this and the install is broken (no trading possible). **High** = lose this and you lose history that's not recoverable from any other source. **Medium** = lose this and you can rebuild from re-running pipelines. **Cache** = drops on `rm` and re-fills on next use.

## Backup strategies

### Minimal — auth + history only (~MB)

If disk is tight or you trust your data to re-sync:

```bash
tar -czf rift-min-backup.tgz \
  ~/.rift/credentials \
  ~/.rift/.env \
  ~/.rift/tokens/ \
  ~/.rift/signal_memory.jsonl \
  ~/.rift/lessons.json \
  ~/.rift/workbench/
```

This is the smallest backup that lets you trade again without re-pairing an agent or losing your trade-outcome learning history.

### Complete — everything except cache (~tens of MB)

If you want a faithful machine snapshot:

```bash
tar -czf rift-full-backup.tgz \
  --exclude='data' \
  --exclude='raw' \
  --exclude='raw_l2' \
  --exclude='collector.db' \
  --exclude='collector.log' \
  ~/.rift/
```

### Archive everything (~GB)

If you don't mind the size and want a one-shot restore:

```bash
tar -czf rift-archive.tgz ~/.rift/
```

Cache regenerates anyway — including it just costs disk + transfer time.

## Moving to a new machine

```bash
# 1. On old machine — back up what matters
tar -czf rift-move.tgz \
  ~/.rift/credentials \
  ~/.rift/.env \
  ~/.rift/tokens/ \
  ~/.rift/signal_memory.jsonl \
  ~/.rift/lessons.json \
  ~/.rift/validated_edge.json \
  ~/.rift/bundles/ \
  ~/.rift/recon/ \
  ~/.rift/algo_sessions/ \
  ~/.rift/workbench/

# 2. Copy to new machine
scp rift-move.tgz user@newmachine:~

# 3. On new machine — fresh install first (see README), then:
mkdir -p ~/.rift
tar -xzf ~/rift-move.tgz -C ~/.rift
chmod 600 ~/.rift/.env ~/.rift/credentials ~/.rift/tokens/*

# 4. Re-download cached data (optional, ~$2 in AWS S3 fees)
rift sync

# 5. Verify
rift doctor
```

The API wallet key in `~/.rift/credentials` remains valid on the new machine — Hyperliquid agent wallets are bound to the main wallet, not the device. Your authorization tokens travel with you.

---

## Permissions hygiene

After any restore, lock down credentials:

```bash
chmod 600 ~/.rift/.env
chmod 600 ~/.rift/credentials
chmod -R 600 ~/.rift/tokens/
```

`rift doctor` will warn if `.env` is world-readable.

---

## Wiping everything

```bash
# Full reset (loses all auth + history + cache)
rm -rf ~/.rift
```

You'll need to re-run `rift auth setup` and reapprove the builder fee. Cached data must be re-synced.

To keep auth but reset signal memory + trade history:

```bash
rm -f ~/.rift/signal_memory.jsonl ~/.rift/lessons.json ~/.rift/validated_edge.json
rm -rf ~/.rift/recon ~/.rift/algo_sessions ~/.rift/bundles
```

## What's safe to commit to a dotfiles repo

**Never:**
- `~/.rift/.env`
- `~/.rift/credentials`
- `~/.rift/tokens/`

These contain private keys and access secrets. If you've already committed them, rotate the keys immediately.

**Probably safe (your own data, no secrets):**
- `~/.rift/lessons.json`
- `~/.rift/workbench/` (your strategy drafts)

**Don't bother:**
- Anything under `data/`, `raw/`, `raw_l2/` — too big and recoverable.
- Anything under `algo/`, `algo_sessions/`, `recon/` — machine-specific state.
